# m4-pr-template-qd-evidence-contract Notes

This node updates the GitHub pull request template so qd-driven PRs consistently
record:

- qd node identity
- acceptance criteria
- validation commands
- evidence paths
- completion and audit report paths
- platform validation status
- residual risks
- DAG or topology changes

The workflow documentation was aligned with the expanded template fields. The
cross-device and opposite-platform fields are marked as relevant only when they
apply to the change.
