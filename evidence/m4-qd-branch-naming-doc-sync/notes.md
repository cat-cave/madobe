# m4-qd-branch-naming-doc-sync Notes

This node aligns qd branch naming instructions with the current qd worktree convention.

The updated guidance tells orchestrators and implementors to use branch names that carry the exact qd node id. New
work should prefer `spec/<node-id>` unless the qd node already records another branch. Historical `qd/` branches are
acknowledged only as existing old records and PRs that may still appear in the repository history.

runtime, capture, encode, transport, decode, render, presentation, cross-device, latency, Mac hardware, workflow
